①問題を追加
・エクセルに問題文、正答、誤答、解説等を追加し上書き保存する
　※セルの入力漏れやズレに注意、音声用の文章も忘れずに記入すること。
②音声ファイルを作成
・コマンドプロンプトでpy create_tts.pyを実行するとmp3が生成される
　※tts_audioフォルダにmp3が追加されたか確認すること
③更新を反映
・コマンドプロンプトでpy quiz_make.pyを実行する
　※question.jsonが更新されていることを確認する(更新日時で可)
④Netlifyにアップロード(GitHub経由でアップする)
・NetlifyとGitHubを両方ログインする
・コマンドプロンプトで以下を実行する
　git status 　←変更確認（任意）変更ファイルが赤や緑で表示される
　git add .
  git commit -m "新しい問題を追加"　←""の中がNetlify上に表示される
  git push origin main
・これで、Netlifyが自動でデプロイされる
　※git pull origin main
　　↑他者と作業を共有する場合はこれをpushの前に実行しておく
　※マージ画面（黒い画面）が出たら、wq と入力して Enterを押す
⑤更新を確認
・Netlifyで公開ページを開き、更新が反映されていることを確認する
　→Netlifyのダッシュボードで対象プロジェクトを開く
　　Deploys タブで「Published」になっていればOK
　※更新が反映されない場合、リロード（Ctrl+F5）も試す

【確認ポイント】
チェック項目	内容
HTML／JSの変更反映	index.html や JavaScript に反映されているか
questions.json 読み込みOKか	コンソールにエラーがないか（例：SyntaxError: Unexpected token < が出ない）
音声が正しく再生されるか	tts_audio/1-01-1.mp3 など外部参照が成功しているか
キャッシュの影響がないか	ブラウザの「スーパーリロード（Ctrl + F5）」を行う

よくあるトラブルと対処法
症状	原因	対処法
git push が rejected される	git pull をしていない	git pull origin main をしてから git push
コンソールに < のSyntaxError	questions.json が正しく出力されていない or path間違い	Netlify/questions.json の中身とURLを確認
音声が再生されない	audioパスが相対参照で誤っている or ファイルがない	tts_audio/1-01-1.mp3 の存在確認、JSで tts_audio/ に正しく参照されているか

補足：Python側の出力パスも確認
以下のように Netlify/ 以下に書き出す構成になっていることが前提です：

output_file = os.path.join(base_dir, "Netlify", "index.html")
questions_json_path = os.path.join(base_dir, "Netlify", "questions.json")
